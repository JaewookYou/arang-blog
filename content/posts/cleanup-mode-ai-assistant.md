## TL;DR

AI 비서에게 `!정리`라고 한 마디만 하면, 자동으로 현재 진행 중인 작업을 분석하고 다음 할 일을 제안받을 수 있다. 이 글에서는 OpenClaw AI 비서와 함께 만든 '정리 모드' 시스템을 소개한다.

---

## 왜 만들었나?

AI 비서와 오래 대화하다 보면 맥락이 흩어지고, 뭘 했는지 뭘 해야 하는지 놓치기 쉽다. 특히 여러 프로젝트를 동시에 진행할 때 더 그렇다.

그래서 **한 마디로 현재 상황을 정리하고 다음 액션을 제안받는 시스템**을 만들었다.

## !정리 모드란?

`!정리`라고 입력하면 AI 비서가 다음을 자동으로 수행한다:

1. **토큰 절약/자동화할 부분 분석** - 반복 작업은 스킬로 만들기
2. **장기기억/지침에 들어갈 내용 정리** - MEMORY.md 업데이트
3. **블로그 글 작성할 부분 탐색** - 트러블슈팅, CTF writeup 등
4. **Git repo 형상관리할 부분 확인** - 커밋 안 된 변경사항
5. **개선 아이디어 제시** - 자가발전 태스크

점검이 끝나면:
- 태스크에 번호를 부여 (1️⃣ 2️⃣ 3️⃣...)
- 이모지로 선택할 수 있게 리액션 추가
- 복수 선택 시 큐잉해서 순차 실행
- 🔥는 전체 실행, ❌는 취소

## 실제 사용 예시

```
나: !정리

AI: 📋 **정리 모드 분석 완료**

1️⃣ **알파헌터 댓글 버그 수정**
- 현재: 댓글이 안 달림
- 원인: author 타입 체크 누락
- 예상 시간: 10분

2️⃣ **에러 핸들링 전수조사**
- 대상: 35개 스크립트
- 부족한 파일: 6개

3️⃣ **MEMORY.md 업데이트**
- 추가할 내용: 오늘 배운 교훈들

🔥 모두 실행 | ❌ 취소
```

나는 그냥 1️⃣ 이모지를 클릭하면 된다. AI가 알아서 작업을 수행하고 결과를 보고한다.

## 웃픈 경험: '!정리' vs `!정리`

처음에 이 기능을 만들 때 재미있는 일이 있었다.

Discord에서 `!정리`라고 치면... **bash script 실행 모드로 빠져버렸다!** 🤦

Discord 봇들이 보통 `!`를 명령어 prefix로 쓰다 보니, OpenClaw도 그렇게 인식한 것이다. 그래서 지금은 **작은따옴표를 앞에 붙여서** `'!정리`로 쓰고 있다.

```
❌ !정리  → bash: !정리: command not found
✅ '!정리 → 정리 모드 실행됨
```

이것도 하나의 UX 교훈이다. AI와 대화할 때는 **특수문자가 어떻게 해석될지** 항상 주의해야 한다.

## 기술적 구현

### MEMORY.md에 정의된 규칙

```markdown
### !정리 모드 (2026-02-01)
**arang이 `!정리`라고 하면 아래 내용 점검:**
1. 토큰 절약/자동화할 부분 (스킬 만들기)
2. 장기기억/지침에 들어갈 내용
3. 블로그 글 작성할 부분
4. Git repo 형상관리할 부분
5. 개선 아이디어

**점검 후:**
- 태스크 리스트 번호로 부여
- 이모지 스킬로 수행할 작업 선택
- 복수 선택 시 큐잉해서 모두 수행
- Git repo는 영어 기본, README_KR.md 별도 작성
```

### 이모지 선택 시스템

Discord의 리액션 기능을 활용한다:
1. AI가 메시지 전송
2. 바로 이모지 리액션 추가 (1️⃣ 2️⃣ 3️⃣ 🔥 ❌)
3. 사용자가 이모지 클릭
4. noti-bot이 선택된 이모지 감지
5. 해당 태스크 실행

## 왜 이게 좋은가?

### 1. 맥락 손실 방지
대화가 길어져도 `!정리` 한 번이면 현재 상태를 파악할 수 있다.

### 2. 의사결정 피로 감소
"뭐 해야 하지?" 대신 AI가 제안한 것 중에서 고르면 된다.

### 3. 작업 추적
어떤 태스크를 선택했는지, 완료했는지 기록이 남는다.

### 4. 점진적 개선
매번 정리할 때마다 시스템이 조금씩 좋아진다. 반복 작업은 자동화되고, 교훈은 기록된다.

## 확장 아이디어

- **정기 정리**: 크론잡으로 하루에 한 번 자동 실행
- **프로젝트별 정리**: 특정 디렉토리나 프로젝트만 분석
- **우선순위 자동 할당**: 긴급도/중요도 기준으로 정렬
- **시간 추정**: 각 태스크에 예상 소요 시간 표시

## 마무리

AI 비서는 명령을 수행하는 도구가 아니라, **함께 일하는 파트너**다. 

`!정리` 모드는 그 파트너와 효율적으로 소통하는 방법 중 하나다. 복잡한 워크플로우를 한 마디로 트리거하고, 결과를 이모지 클릭 하나로 선택할 수 있다.
